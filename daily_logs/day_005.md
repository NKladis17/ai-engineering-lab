# Day 5 — Prompt Regression Testing (first evals)

## Built
- prompt_regression_test.py: scores 2 prompts against fixed test cases
- Scoring rule: keyword presence AND brevity (≤25 words)

## Learned
- Evals replace "looks good" with a defensible number
- 5 load-bearing parts: fixed test cases, per-case rules, prompt as ONLY variable (temp 0), automatic identical scoring, loop+tally
- A too-easy test set can't discriminate (first run: 5/5 tie)
- A bad threshold makes good answers fail (15 words was too strict)
- The hard part of evals is calibrating the measuring stick, not the prompts
- score() uses AND: an answer must meet ALL quality bars, not just one

## Evidence
- Calibrated result: Prompt A (vague) 0/5, Prompt B (specific) 5/5
- Took 3 threshold iterations to get an honest measurement
