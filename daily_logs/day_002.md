# Day 2 — Structured Outputs + JSON Validation

## Built
- structured_logger.py: forces model to return JSON, validates with Pydantic
- StructuredAnswer schema (summary, key_points, confidence 0–1, possible_errors)
- Defensive fence-stripping to handle markdown backticks
- JSONL logging of raw + parsed output with pass/fail status

## Learned
- The model is probabilistic — it broke the "no backticks" rule on call #1
- Two failure types: JSON decode (not parseable) vs Pydantic (wrong shape)
- "Nice answer" ≠ "valid machine-readable output"
- You don't trust model output — you inspect, clean, and validate it (inspector at the door)

## Bugs / Confusions
- First call failed: model wrapped JSON in markdown fences despite instructions
- Got stuck in dquote> from a missing closing quote (Ctrl+C escapes it)

## Evidence
- Valid structured calls: 5
- Failed validation calls: 2 (1 fence, 1 deliberate weak-prompt test)
- Both failures were JSON decode errors
- Most common failure: model adding markdown formatting
